# Content Design Starter Template

A comprehensive resource for content designers, featuring practical templates, workflows, and guidelines for creating clear, accessible, user-focused content.

## What's Inside

### Templates
Ready-to-use templates for common content design challenges:

**Messages**
- [Error Messages](templates/messages/error-messages.md) - Write clear, actionable error messages
- [Success Messages](templates/messages/success-messages.md) - Confirm actions effectively
- [Empty States](templates/messages/empty-states.md) - Guide users when content is missing
- [Microcopy](templates/messages/microcopy.md) - Button labels, links, and UI text

**Flows**
- [Onboarding](templates/flows/onboarding.md) - Design effective first-time experiences
- [Confirmation Dialogs](templates/flows/confirmation-dialogs.md) - Prevent mistakes with clear confirmations

**Documentation**
- [Release Notes](templates/documentation/release-notes.md) - Write user-focused release notes

### Workflows
Practical guides for content design processes:
- [Git Workflow for Content Designers](workflows/git-workflow.md) - Version control basics
- [Content Review Process](workflows/content-review-process.md) - Structured review and approval

### Resources
Essential guidelines and references:
- [Accessibility Guidelines](resources/accessibility-guidelines.md) - Create content everyone can use

## Quick Start

1. **Clone this repository**
   ```bash
   git clone https://github.com/jkelleman/content-design-starter-template.git
   ```

2. **Browse templates** in the `templates/` folder

3. **Copy and customize** templates for your project

4. **Follow workflows** for Git and review processes

5. **Check accessibility** using the guidelines

## Who This Is For

- Content designers new to VS Code and Git
- Writers transitioning to product content
- UX designers writing interface copy
- Product teams creating content templates
- Anyone writing user-facing content

## How to Use This Template

**For a specific content challenge:**
Navigate to the relevant template and follow the framework

**For a new project:**
1. Start with the [Content Design Starter](content_design_starter.md)
2. Reference specific templates as needed
3. Follow the [Content Review Process](workflows/content-review-process.md)

**For team adoption:**
1. Share the [Git Workflow](workflows/git-workflow.md) with your team
2. Establish your review process
3. Customize templates to match your style guide
